## Description

En répondant aux besoins les plus courants, les bibliothèques JavaScript telles que jQuery et les frameworks CSS prêts à l’emploi (Bootstrap, Skeleton, Gumby, Foundation...) permettent de réaliser rapidement des sites. Mais tous ne s’appuient pas sur une architecture modulaire, ce qui contraint l’internaute à télécharger toute la bibliothèque pour n’utiliser qu’un faible pourcentage des fonctionnalités.

Il est préférable de se passer de ces bibliothèques ou de n’en conserver que les portions réellement utilisées. Utiliser un bundler (par exemple, Webpack) permet de faire facilement du tree shaking, c’est-à-dire d’éliminer du code mort et donc inutilisé. On peut aussi estimer le poids d’une bibliothèque avec le site [https://bundlephobia.com/](https://bundlephobia.com/) qui vous donnera une idée du poids total d’un fichier «package.json».

## Exemple

Certains frameworks comme Tailwind CSS proposent des bibliothèques allégées qui ne contiennent que les portions réellement utilisées. Cela réduit le poids des fi chiers téléchargés par le navigateur et les ressources consommées lors de l’exécution. Par exemple, jQuery propose une version « slim » qui n’inclut pas certaines fonctionnalités comme les animations CSS et le support des échanges de données en AJAX. Cela permet d’économiser 17 Ko pour la version 3.6.
